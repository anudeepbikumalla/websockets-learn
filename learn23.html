The Core Challenge ‚Äî Conflict Resolution üîÄ
Two users type at the same time. Whose changes win?
The conflict problem
// Document: "Hello"
// Alice inserts " World" at position 5 ‚Üí "Hello World"
// Bob inserts "!" at position 5 (simultaneously) ‚Üí "Hello!World"?

// Problem: Alice and Bob's operations based on position 5
// but the document state changed between their edits
// Solution: transform Bob's position AFTER accounting for Alice's insert
// Bob's "!" should now go to position 11 ‚Üí "Hello World!"

This is what Operational Transformation (OT) solves ‚Äî transforming operations to account for concurrent edits.

Simple OT ‚Äî Insert/Delete Operations üíª
The minimal OT operations: insert and delete
</> ot.js ‚Äî basic OT transform
// Operation: { type, pos, char, revision }
function transform(op, against) {
  if (op.type === 'insert' && against.type === 'insert') {
    if (against.pos <= op.pos) op.pos++; // shift right
  }
  if (op.type === 'insert' && against.type === 'delete') {
    if (against.pos < op.pos) op.pos--; // shift left
  }
  return op;
}

// Server: maintain revision history, transform against missed ops
function applyOp(doc, op) {
  if (op.type === 'insert')
    return doc.slice(0, op.pos) + op.char + doc.slice(op.pos);
  if (op.type === 'delete')
    return doc.slice(0, op.pos) + doc.slice(op.pos + 1);
}

‚úÖ Collaborative Editing Rules

üî¢
Track revision numbers ‚Äî every operation includes the revision it was based on. The server transforms it against any operations applied since that revision.
üì¶
Use Yjs or Automerge ‚Äî don't implement OT from scratch for production. Yjs (CRDT-based) works offline and is used by Slate, Quill, and CodeMirror.
üîå
Yjs + WebSocket = 3 lines ‚Äî y-websocket is Yjs's official WebSocket provider. Awareness (presence + cursors) is built-in.
üíæ
Persist the document state ‚Äî store the Yjs document update in Redis or a database so it survives server restarts.
