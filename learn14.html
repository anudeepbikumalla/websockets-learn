Shared Express + WS Server ‚ö°

In real apps, WebSocket and HTTP live on the same port. Learn how to mount a WebSocket server on an existing Express app ‚Äî sharing port 3000 for both REST and real-time.

1
The Problem ‚Äî two separate servers
Real apps share one port for HTTP and WebSocket
‚ùå Separate servers (tutorial style)
// HTTP on :3000, WS on :8080 ‚Äî TWO ports
app.listen(3000);
new WebSocketServer({ port: 8080 });

Two ports = two firewall rules, two SSL certs. Don't do this.

‚úÖ Shared server (production style)
// Both on :3000 ‚Äî ONE port
const server = http.createServer(app);
const wss = new WebSocketServer({ server });
server.listen(3000);

One port, one SSL cert, professional.

2
Full Setup ‚Äî complete server.js üíª
Express REST API + WebSocket on the same port 3000
</> server.js ‚Äî complete shared server
const express = require('express');
const http = require('http');
const { WebSocketServer } = require('ws');

const app = express();
app.use(express.json());

// REST routes (HTTP)
app.get('/api/status', (req, res) => {
  res.json({ clients: wss.clients.size });
});
app.post('/api/broadcast', (req, res) => {
  wss.clients.forEach(c => c.readyState === 1 && c.send(req.body.msg));
  res.json({ sent: wss.clients.size });
});

// Shared HTTP server
const server = http.createServer(app);
const wss = new WebSocketServer({ server });

wss.on('connection', (ws, req) => {
  // req.socket, req.headers, cookies ‚Äî all available!
  ws.send(JSON.stringify({ type: 'welcome' }));
});

server.listen(3000, () => console.log('Port 3000'));
Key: Pass the HTTP server object into WebSocketServer, not a port number. Call server.listen() once ‚Äî it handles both HTTP and WS.
3
Path-based WS routing üîÄ
Multiple WebSocket endpoints on the same port
</> Route /ws and /admin separately
const wss = new WebSocketServer({ noServer: true });
const admin = new WebSocketServer({ noServer: true });

server.on('upgrade', (req, socket, head) => {
  if (req.url === '/ws')
    wss.handleUpgrade(req, socket, head, ws => wss.emit('connection', ws, req));
  else if (req.url === '/admin')
    admin.handleUpgrade(req, socket, head, ws => admin.emit('connection', ws, req));
  else
    socket.destroy(); // reject unknown paths
});

üèóÔ∏è Scenario Implementation: Sharing an HTTP Port with Express
The Design Approach
Often organizations only expose port 80/443 to the internet. You cannot run Express on port 3000 and WebSocket on port 8080. By creating a raw Node `http.Server`, providing it to Express, and attaching the WebSocket server to the SAME HTTP instance, they intelligently share the port (HTTP handles regular routes, WS handles the Upgrade).
Implementation Code
const express = require('express');
const http = require('http');
const WebSocket = require('ws');

const app = express();
app.get('/api/health', (req, res) => res.send({ status: 'OK' }));

// 1. Create a raw server wrapping Express
const server = http.createServer(app);

// 2. Attach WebSocket to the *same* raw server
const wss = new WebSocket.Server({ server });

wss.on('connection', (ws) => {
  ws.send('Connected to unified server!');
});

// 3. Listen on the shared server, NOT app.listen()
server.listen(443, () => console.log('Listening on 443'));

üí° Key Takeaways for Beginners
‚ñ∏By default, Express and WebSocket run on different ports. But you can share one port for both!
‚ñ∏Create the HTTP server from Express using http.createServer(app), then pass it to WebSocket.Server.
‚ñ∏This means your REST API and WebSocket server are one deployment ‚Äî simpler to manage and deploy.
‚ñ∏Use REST for CRUD operations, WebSocket for real-time updates. Best of both worlds.
‚ñ∏This is the pattern used by most production apps ‚Äî a REST API with a WebSocket upgrade endpoint.
üíª Express + WebSocket on same port
const express = require('express');
const http = require('http');
const WebSocket = require('ws');

const app = express();
const server = http.createServer(app); // Create HTTP server from Express

// Attach WebSocket to the SAME server
const wss = new WebSocket.Server({ server });

// REST endpoints (normal Express)
app.get('/api/status', (req, res) => {
  res.json({ status: 'ok', wsClients: wss.clients.size });
});

// WebSocket handler
wss.on('connection', (ws) => {
  ws.send('Hello from shared server!');
});

// ONE listen() for both HTTP and WebSocket
server.listen(8080, () => {
  console.log('HTTP + WS running on port 8080');
});
