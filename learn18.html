Offline Queue üì¨

Users go offline. Network connections drop. An offline queue buffers messages while disconnected and automatically flushes them when the connection is restored.

1
The Problem ‚Äî messages lost on disconnect ‚ùå
Without a queue, any send() call while offline silently fails
‚ùå Without offline queue
// User goes offline, then tries to send
ws.send('Hello!'); // üíÄ throws if ws is closed
// readyState === 3 (CLOSED) ‚Üí message dropped silently

Mobile users switching towers, laptops sleeping, intermittent WiFi ‚Äî all cause this.

2
The Solution ‚Äî a send() wrapper with a queue ‚úÖ
Queue messages when offline, flush them when reconnected
</> client.js ‚Äî offline queue wrapper
let ws, queue = [];

// Safe send ‚Äî queues if offline
function safeSend(msg) {
  if (ws && ws.readyState === WebSocket.OPEN) {
    ws.send(msg);
  } else {
    queue.push(msg); // buffer for later
    console.log(`Queued (${queue.length} pending)`);
  }
}

function connect() {
  ws = new WebSocket(getWsUrl(8080));
  ws.addEventListener('open', () => {
    // Flush all queued messages on reconnect
    while (queue.length) ws.send(queue.shift());
    console.log('Queue flushed!');
  });
  ws.addEventListener('close', () => setTimeout(connect, 3000));
}

‚úÖ Offline Queue Rules

üîÅ
Replace ws.send() with safeSend() throughout your codebase ‚Äî never call the raw WebSocket send directly.
üìè
Cap queue size ‚Äî limit to ~100 messages. If the queue overflows, drop the oldest messages to prevent memory leaks.
üíæ
Persist to localStorage ‚Äî for PWAs, persist the queue to localStorage so messages survive page refreshes.
üîÑ
Combine with reconnect ‚Äî pair with Lesson 5's exponential backoff. The queue flushes automatically on reconnect.
üåç Offline Queue ‚Äî Who Uses This Pattern?
üí¨
WhatsApp
Messages sent while offline (airplane mode, no signal) are queued on your phone. When connectivity returns, they're flushed in order with correct timestamps.
üìß
Gmail Offline
Gmail's offline mode queues outgoing emails in IndexedDB. When you reconnect, it syncs ‚Äî sending queued emails and downloading new ones.
üìù
Notion / Google Docs
Edits made while offline are queued locally. When you reconnect, they're synced using conflict resolution (OT or CRDT) to merge with others' changes.
üó∫Ô∏è
Google Maps (Offline)
Navigation directions downloaded offline queue location updates. When signal returns, your actual route is synced back to Google.

üí° Key Takeaways for Beginners
‚ñ∏Users will go offline ‚Äî WiFi drops, subway tunnels, laptop lid closes. Their messages should not be lost.
‚ñ∏An offline queue buffers messages in an array. When the connection comes back, flush (send) the entire queue.
‚ñ∏Check ws.readyState === WebSocket.OPEN before sending. If not open, push to the queue instead.
‚ñ∏Set a max queue size (e.g., 100 messages) to avoid memory issues if the user is offline for a long time.
‚ñ∏For critical apps, also save the queue to localStorage so it survives page refreshes.
üíª Offline message queue
const queue = []; // Messages waiting to be sent
const MAX_QUEUE = 100;

function send(data) {
  const message = JSON.stringify(data);

  if (ws && ws.readyState === WebSocket.OPEN) {
    ws.send(message);  // Online ‚Äî send immediately
  } else {
    // Offline ‚Äî save to queue (with size limit)
    if (queue.length < MAX_QUEUE) {
      queue.push(message);
      console.log(`üì¶ Queued (${queue.length} waiting)`);
    }
  }
}

// When connection comes back, flush the queue
ws.addEventListener('open', () => {
  console.log(`üîÑ Flushing ${queue.length} queued messages...`);
  while (queue.length > 0) {
    ws.send(queue.shift());  // Send oldest first
  }
});
