The Signaling Flow ‚Äî Step by Step üîÑ
6 steps from "I want to call Bob" to peer connection
üìã Signaling sequence
1
Alice clicks "Call Bob" ‚Äî both connect to the signaling server via WebSocket
2
Alice creates an RTCPeerConnection and generates an SDP Offer (describes her audio/video capabilities)
3
Alice sends the SDP Offer to the signaling server ‚Äî WS relays it to Bob
4
Bob creates an RTCPeerConnection, reads the offer, and generates an SDP Answer
5
Both sides exchange ICE Candidates (network paths) via WebSocket ‚Äî the browser discovers them automatically
6
WebRTC peer connection established! Video/audio flows directly p2p ‚Äî WebSocket is no longer needed

const rooms = new Map(); // roomId ‚Üí [ws1, ws2]

wss.on('connection', (ws) => {
  ws.on('message', (raw) => {
    const msg = JSON.parse(raw);

    if (msg.type === 'join') {
      const room = rooms.get(msg.room) || [];
      room.push(ws); rooms.set(msg.room, room);
      ws.room = msg.room;
      if (room.length === 2) // 2nd peer joined ‚Äî start signaling
        room[0].send(JSON.stringify({ type: 'start' }));
    }
    // Relay SDP/ICE to the other peer in the room
    if (['offer','answer','ice'].includes(msg.type)) {
      const peers = rooms.get(ws.room) || [];
      peers.filter(p => p !== ws).forEach(p => p.send(raw.toString()));
    }
  });
});
The server just relays ‚Äî it never looks inside the SDP or ICE candidates. It's a simple message relay between two peers in the same room.

‚úÖ WebRTC + WebSocket Rules

üì°
WebSocket is only for signaling ‚Äî once the RTCPeerConnection is established, data flows p2p. Close the WS if you no longer need server communication.
üåç
You need a TURN server for NAT traversal ‚Äî most corporate firewalls block direct p2p. TURN servers relay traffic as a fallback. Use Twilio or Metered.ca for free TURN.
üì¶
Use PeerJS or simple-peer for production ‚Äî they wrap the WebRTC API with a simple peer.connect(id) / peer.on('data') interface.

üí° Key Takeaways for Beginners
‚ñ∏WebRTC = direct peer-to-peer connections (no server in between). Perfect for video/voice calls.
‚ñ∏But peers need to find each other first ‚Äî that's what WebSocket signaling does (exchange connection details).
‚ñ∏The flow: WebSocket connects both users ‚Üí they exchange SDP offers/answers ‚Üí ICE candidates find the route ‚Üí P2P connection established.
‚ñ∏After the P2P connection is established, the WebSocket signaling server is no longer needed.
‚ñ∏WebSocket = signaling (small text messages). WebRTC = media (video/audio streams). They work together!
