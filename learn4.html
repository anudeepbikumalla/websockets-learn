üéß Receiving messages ‚Üí Lesson 2

We use addEventListener (not onmessage) so multiple listeners can react ‚Äî the chat UI updates AND a title counter could also update.

// Lesson 2 ‚Äî addEventListener, not onmessage
ws.addEventListener('message', (e) => {
  // Handler A: show message in chat
  appendBubble(e.data, 'server');
});
// Could add a 2nd listener too ‚Äî both fire!
üîÑ Sending messages ‚Üí Lesson 3

The send button uses async/await style: the message goes out, the server processes it, and the response comes back asynchronously via the message event.

// Lesson 3 ‚Äî async, no callback needed
async function chatSend() {
  const text = input.value;
  // Fire and forget ‚Äî reply comes via addEventListener
  ws.send(JSON.stringify({ type: 'echo', data: text }));
}

‚úÖ Complete WebSocket Knowledge

1
What it is: A permanent 2-way phone call between browser and server ‚Äî unlike HTTP which is a one-off letter.
2
4 lifecycle stages: new WebSocket() ‚Üí open event ‚Üí send/receive ‚Üí close event. Always in this order.
3
addEventListener vs onmessage: Use addEventListener in real apps ‚Äî it lets multiple handlers coexist. onmessage = silently kills any previous handler.
4
Async patterns: Use async/await on the server for clean, readable processing. Callbacks work but lead to deeply nested "callback hell" code.
5
Real uses: Chat apps, live scores, multiplayer games, stock tickers, Google Docs live editing, AI token streaming ‚Äî any time data needs to flow in real-time.

üèóÔ∏è Scenario Implementation: Customer Support Messaging Pattern
The Design Approach
Real-world chat apps don't send plain strings like "Hello". They send structured JSON payloads. This allows the receiver to instantly know exactly *who* sent the message, *when* it was sent, what *room* it belongs in, and if it's a text message or a file attachment.
Implementation Code
// 1. Structure the message
const payload = {
  type: "CHAT_MESSAGE",
  roomId: "support-ticket-109",
  sender: {
    id: "user-44",
    role: "customer"
  },
  content: "I need help with my refund.",
  timestamp: new Date().toISOString()
};

// 2. Serialize to string and send
ws.send(JSON.stringify(payload));

üí° Key Takeaways for Beginners
‚ñ∏A chat app combines all three lessons: connection (L1), event listeners (L2), and async patterns (L3).
‚ñ∏Always send JSON objects instead of plain strings ‚Äî it lets you add metadata like username, timestamp, message type.
‚ñ∏Disable the send button when not connected ‚Äî check ws.readyState === WebSocket.OPEN before sending.
‚ñ∏Show connection status to the user (green dot = connected, red = disconnected) ‚Äî never leave them guessing.
‚ñ∏This is your first real WebSocket app! Every chat app (WhatsApp, Slack, Discord) uses this exact same pattern at its core.
üíª Chat app core pattern
const ws = new WebSocket('ws://localhost:8080');

// Show connection status to the user
ws.addEventListener('open', () => {
  statusDot.style.background = 'green'; // Visual feedback!
  sendButton.disabled = false;          // Enable sending
});

// Display incoming messages
ws.addEventListener('message', (event) => {
  const msg = JSON.parse(event.data);  // Always parse JSON
  addMessageToChat(msg.text, 'server'); // Add to chat UI
});

// Send button click handler
async function sendMessage() {
  const text = input.value.trim();
  if (!text || ws.readyState !== WebSocket.OPEN) return;

  // Send as structured JSON (not plain text!)
  ws.send(JSON.stringify({
    type: 'chat',           // Message type
    text: text,             // The actual message
    timestamp: Date.now()   // When it was sent
  }));

  addMessageToChat(text, 'mine');  // Show in our chat
  input.value = '';                // Clear the input
}
