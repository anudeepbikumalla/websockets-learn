"Alice is typing‚Ä¶" ‚Äî one of the most recognizable real-time UX patterns. Learn how to implement it correctly with debouncing to avoid message spam.

The naive approach ‚Äî and why it's wrong ‚ùå
Sending a typing event on every keypress floods the server
‚ùå Bad ‚Äî fires on every keypress
input.addEventListener('keydown', () => {
  // 100WPM typist = ~10 events/sec üî•
  ws.send(JSON.stringify({ type: 'typing' }));
});

A fast typist sends 600 events/minute. The server broadcasts all of them. This is wasteful and gets rate-limited.

2
The right way ‚Äî debounced typing + stop timer ‚úÖ
Send "typing" once, then "stopped" after 2s of inactivity
</> client.js ‚Äî debounced typing
let typingTimer = null;
let isTyping = false;

input.addEventListener('input', () => {
  if (!isTyping) {
    isTyping = true;
    ws.send(JSON.stringify({ type: 'typing', username }));
  }
  // Reset the stop timer on every keystroke
  clearTimeout(typingTimer);
  typingTimer = setTimeout(() => {
    isTyping = false;
    ws.send(JSON.stringify({ type: 'stopped', username }));
  }, 2000); // stop after 2s inactivity
});
</> server.js ‚Äî broadcast typing events
const typing = new Set(); // set of usernames currently typing

ws.on('message', (raw) => {
  const { type, username } = JSON.parse(raw);
  if (type === 'typing') typing.add(username);
  if (type === 'stopped') typing.delete(username);
  broadcast({ type: 'typing_update', users: [...typing] });
});

‚úÖ Typing Indicator Rules

‚è≥
Always debounce ‚Äî send "typing" once per session, not per keystroke. Use a 2s stop timer.
üì§
Send "stopped" on message send ‚Äî when the user hits Enter and sends a message, immediately broadcast stopped without waiting for the 2s timer.
üë•
Handle multiple typers ‚Äî use a Set on the server. UI should show "Alice and Bob are typing‚Ä¶" when both are active.
üí§
Clear timers on disconnect ‚Äî if a user disconnects while typing, remove them from the typing Set and broadcast the update.
ü§ñ
ChatGPT
The "thinking" animation and token-by-token text appearance is a form of typing indicator ‚Äî showing the AI is actively generating a response.
üì±
Instagram DMs
"Seen" receipts and typing indicators use WebSocket. Instagram sends typing events to the chat partner's device only, not to all users.
üè¢
Intercom / Zendesk
Customer support agents see "Customer is typing..." which helps them wait before sending their next response, creating a more natural conversation flow.

üí° Key Takeaways for Beginners
‚ñ∏Typing indicators show "Alice is typing..." in real-time. This makes conversations feel alive.
‚ñ∏Debouncing is critical ‚Äî don't send a typing event for every keystroke. Wait until the user pauses for 300ms.
‚ñ∏Auto-clear the indicator after 3 seconds of silence. Otherwise it stays forever if the user stops and doesn't send.
‚ñ∏Only broadcast typing to the relevant room/conversation ‚Äî not to all users globally.
‚ñ∏This is a simple but powerful UX pattern ‚Äî it tells the other person to wait before leaving the chat.

    ws.send(JSON.stringify({
      type: 'typing',
      user: myUsername

// CLIENT ‚Äî Show and auto-clear typing indicator
ws.addEventListener('message', (event) => {
  const msg = JSON.parse(event.data);
  if (msg.type === 'typing') {
    showTyping(msg.user); // Display "Alice is typing..."

    // Clear after 3 seconds if no more typing events
    clearTimeout(window.typingClearTimer);
    window.typingClearTimer = setTimeout(() => {
      hideTyping(msg.user);
    }, 3000);
  }
});
