      client.send(data);

// When a WS client sends a message ‚Üí publish to Redis

  pub.publish('messages', raw.toString());

The magic: Every server publishes to Redis and receives from Redis. So a message from Alice (Server A) is delivered to all servers, and each server forwards it to its own local WebSocket clients.

üèóÔ∏è Scenario Implementation: Horizontal Scaling with Redis
The Design Approach
A single Node.js process maxes out around 65,000 connections. To build an app like WhatsApp, you need dozens of servers. But if Alice is on Server A and Bob is on Server B, Server A can't directly interact with Bob's socket. We solve this by adding Redis Pub/Sub as the central 'post office' connecting all node servers.
Implementation Code
const Redis = require('ioredis');
const redisSub = new Redis();
const redisPub = new Redis();

// 1. Subscribe to the global 'chat_messages' channel
redisSub.subscribe('chat_messages');

// 2. When ANY server publishes a message, broadcast it to our LOCAL clients
redisSub.on('message', (channel, payload) => {
  if (channel === 'chat_messages') {
    wss.clients.forEach(c => c.send(payload));
  }
});

// 3. When a local client speaks, publish it globally for other servers
wss.on('connection', (ws) => {
  ws.on('message', (msg) => {
    redisPub.publish('chat_messages', msg);
  });
});

üí° Key Takeaways for Beginners
‚ñ∏A single server can handle ~10K-65K WebSocket connections. Beyond that, you need multiple servers.
‚ñ∏The problem: User A on Server 1 sends a message, but User B is on Server 2. Server 2 doesn't know about the message.
‚ñ∏Redis Pub/Sub solves this ‚Äî every server subscribes to a shared channel. When one server gets a message, it publishes to Redis, and all servers receive it.
‚ñ∏Sticky sessions ensure a user's requests always go to the same server ‚Äî important because WebSocket is stateful.
‚ñ∏In production, use managed services like AWS ElastiCache (Redis) and ALB (load balancer) to handle this automatically.

const sub = new Redis();  // Subscriber

    if (client.readyState === WebSocket.OPEN) {
      client.send(message);  // Send to our local clients
    }
  });

// When a local client sends a message...

    // Publish to Redis ‚Äî ALL servers will receive this

  });
});
