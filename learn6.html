The Server Code üíª
Server keeps a list of all clients and loops through them
</> server.js ‚Äî broadcast to all clients
const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 8080 });

// wss.clients = a Set of ALL connected sockets
wss.on('connection', (ws) => {
  ws.on('message', (raw) => {
    const msg = raw.toString();

    // ‚îÄ‚îÄ The broadcast loop ‚îÄ‚îÄ
    wss.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(JSON.stringify({
          type: 'broadcast',
          from: 'anonymous',
          text: msg
        }));
      }
    });
  });
});

üí° Key insight: wss.clients is a live Set ‚Äî it auto-updates as clients connect and disconnect. Always check readyState === OPEN before sending or you'll get errors.

Broadcast Variants üéõÔ∏è
Broadcast to all, or everyone except the sender
üì¢ Send to ALL (including sender)
wss.clients.forEach((client) => {
  if (client.readyState === WebSocket.OPEN) {
    client.send(msg);
  }
});

Used for: live score tickers, announcement boards, server-initiated pushes.

üì§ Send to OTHERS (exclude sender)
wss.clients.forEach((client) => {
  if (client !== ws &&
      client.readyState === WebSocket.OPEN) {
    client.send(msg);
  }
});

Used for: chat apps where sender sees their own message immediately without an echo.

‚úÖ Broadcast Checklist

‚òëÔ∏è
Always check readyState === OPEN before sending ‚Äî a client in CLOSING/CLOSED state will throw an error.
‚ö°
wss.clients is a live Set ‚Äî you don't need to manage it yourself. Clients are added on connect and removed on disconnect automatically.
üìè
Broadcast scales linearly ‚Äî 1000 clients = 1000 send() calls per message. For very high traffic, consider a message broker like Redis Pub/Sub.
üè∑Ô∏è
Include metadata ‚Äî always send { type, from, text, timestamp } so clients can display and filter messages correctly.
