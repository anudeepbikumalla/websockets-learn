Message History & Replay üìú

When a user reconnects, they shouldn't miss what happened. Learn how to store recent messages on the server and replay them on every new connection.

1
Server ‚Äî ring buffer for recent messages üíª
Store the last N messages, replay on connect
</> server.js ‚Äî history with ring buffer
const MAX_HISTORY = 50;
const history = []; // ring buffer ‚Äî last 50 messages

wss.on('connection', (ws) => {
  // ‚ë† Replay history to this new client
  ws.send(JSON.stringify({
    type: 'history',
    messages: history
  }));

  ws.on('message', (raw) => {
    const msg = { ...JSON.parse(raw), ts: Date.now() };

    // ‚ë° Save to history (keep only last 50)
    history.push(msg);
    if (history.length > MAX_HISTORY) history.shift();

    // ‚ë¢ Broadcast to everyone
    wss.clients.forEach(c => c.readyState === 1 && c.send(JSON.stringify(msg)));
  });
});

‚úÖ Message History Rules

üìè
Cap history size ‚Äî 50‚Äì100 messages in memory is fine. For unlimited history, use a database (Redis, Postgres) and load on demand.
üïí
Always store timestamps ‚Äî clients need ts to show relative times ("3 minutes ago") and to know if messages are stale.
üîå
Send history on connect, not on request ‚Äî push history immediately on connection so users see it without a round-trip.
üé®
Visually distinguish history ‚Äî use a different color or opacity for replayed messages so users know what they missed vs what's new.

üí° Key Takeaways for Beginners
‚ñ∏New users joining a chat should see the recent conversation, not an empty screen. This is message replay.
‚ñ∏A ring buffer is a fixed-size array that overwrites the oldest message when full. Perfect for "last N messages".
‚ñ∏When a client connects, send them the buffered messages first, styled differently (grayed out), then a divider line.
‚ñ∏For production, use a database (PostgreSQL) instead of in-memory arrays ‚Äî they survive server restarts.
‚ñ∏Mark historical messages differently from live ones so users can tell what's old and what's new.
üíª Message replay on join
const HISTORY_SIZE = 50;
const messageHistory = []; // Ring buffer

wss.on('connection', (ws) => {
  // Step 1: Send recent history to the new client
  ws.send(JSON.stringify({
    type: 'history',
    messages: messageHistory.slice(-HISTORY_SIZE)
  }));

  // Step 2: Handle new messages
  ws.on('message', (data) => {
    const msg = JSON.parse(data);
    msg.timestamp = Date.now();

    // Add to history (ring buffer ‚Äî keep only last N)
    messageHistory.push(msg);
    if (messageHistory.length > HISTORY_SIZE) {
      messageHistory.shift(); // Remove oldest
    }

    // Broadcast to all clients as a live message
    broadcast(JSON.stringify({ type: 'live', ...msg }));
  });
});
